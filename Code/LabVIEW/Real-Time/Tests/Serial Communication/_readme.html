Robot Communications using Network Streams

This template shows how you could use Network Streams to pass information between processes on a robot.  The processes could all be running on the same machine, or some could be running on a different machine on the network.  This example will work for desktop and RT targets.

See the help file on robot data communications for additional information.

Running the example on your PC

Open and run the following VI's in any order:

  • Read write template.vi
  • Write template.vi
  • Read template.vi

Running the example on an RT target

  1. Add the RT target you want to use in the Project Explorer window of your project. If prompted, choose to use the FPGA Interface mode.
  2. Drag the VIs you want to run on the RT Target from "My Computer" to the RT Target you added in the last step.
  3. The Read template.vi points to the Read write template.vi in order to read data.  The Read write template.vi points to the Write template.vi in order to read a different type of data.  After moving VIs to the target you want them to run on, replace [localhost] in the Data * writer endpoint URL control on Read template.vi and Read write template.vi to point to the correct IP address the data is being sent from.
  4. Run the VIs in any order.

Required software: Network Streams installed on the RT target (can done by adding software through MAX).
